# **晚礼服 B2B 官网 UI 设计规范 (UI Guide) - Ver 3.0**

**版本代号**：Midnight Architect / 午夜建筑师

**核心理念**：Industrial Precision (工业级精度) & The Archive (档案库)

**关键词**：日系冷调、锐利几何 (Sharp Geometry)、信息高密、秩序感、截屏友好

本规范旨在响应 B2B 批发业务对“稳定、高效、专业”的诉求。我们将摈弃所有装饰性的柔和元素（如圆角、弥散阴影、暖色光晕），转而采用一种类似于“建筑蓝图”或“金融终端”的冷峻美学。网站不再是一个展示厅，而是一个展示制造能力与供应链权威的数字档案库。

---

## **1. 信息架构与布局逻辑 (IA & Layout)**

### **1.1 顶层隐喻：从“画廊”到“档案库”**

* **旧版隐喻**：画廊 (Gallery) —— 强调慢节奏、留白与呼吸感。
* 
**新版隐喻**：**档案库 (The Archive)** —— 强调检索效率、高密度信息展示与工业秩序 。


* **首页**：制造能力的“大银幕”与实时资讯的“股票板”。
* **列表页**：类似于接触印相 (Contact Sheet) 或 Excel 表格的高效浏览视图。
* **详情页**：标准化的工业规格单 (Spec Sheet)。



### **1.2 网格系统：紧凑型模块化网格 (Compact Modular Grid)**

放弃原本宽间距的布局，采用高密度的网格系统。

* **间距 (Gaps)**：
* **Desktop**：`20px` (甚至 `0px`，仅用 `1px` 分割线区分)。
* **Mobile**：`8px`。
* 
**目的**：最大化屏幕利用率，允许批发商在单屏内浏览更多款式 。




* **结构节奏**：
* **Header**：全宽 (Full-width) 沉浸式。
* **Body**：`5-Column` 或 `6-Column` 瀑布流 (Masonry)，打破常规 B2C 网站 3 列的限制。

## **2. 首页核心板块设计 (Home Sections)**

### **2.1 Hero Section：制造剧场 (Production Theatre)**

* 
**视觉目标**：建立“源头工厂”的信任感与“日系冷调”的品牌基调 。


* **内容构成**：
* **背景**：100vh 全屏循环视频。内容为工厂实拍（针脚特写、剪裁瞬间、面料质感），必须叠加 **“Cold Cyan Filter (青冷滤镜)”**。
* **前景文字**：不使用大标题，而是使用类似于“电影字幕”或“代码注释”的排版。
* 位置：左下角或右下角。
* 字体：等宽字体 (Monospace)，纯白。
* 文案：“自2008年建厂... 20年的制衣匠心” 。






* **交互**：无多余动效，仅保留视频本身的蒙太奇剪辑节奏。

### **2.2 资讯传真 (The Ticker)**

* **形态**：屏幕下方或导航栏下方的一条窄通栏。
* **视觉**：背景色为品牌色 **Midnight Abyss (#000226)**，文字为纯白。
* **功能**：类似于股票行情或新闻滚动的 **Marquee (跑马灯)** 效果。
* 
**内容**：展会预告、放假通知、新款发布 。


* **隐喻**：传递商业信息的实时性与紧迫感。

### **2.3 矩阵货架 (The Grid - Core)**

* **排版**：高密度瀑布流。
* **展示逻辑**：
* 
**默认状态**：模特全身图 (Cutout/去底 或 纯白背景) 。


* 
**悬停状态 (Hover)**：**瞬间切换 (0s delay)** 至面料微距特写或背部细节视频 。


* **数据外显**：每张图下方直接展示 `Style No. (款号)` 和 `Wholesale Price (批发价 - 登录可见)`，使用 **DIN** 或 **JetBrains Mono** 字体，模拟价签。

## **3. 组件与原子设计系统 (Atomic Design)**

### **3.1 形状与材质：锐利几何 (Radical Geometry)**

* 
**圆角 (Border Radius)**：全局 **0px**。无论是按钮、图片容器还是输入框，必须是绝对的直角 。


* *理由*：圆角代表亲和与妥协，直角代表严谨与工业标准。


* **边框 (Borders)**：
* **宽度**：`1px` 实线。
* **颜色**：默认浅灰 `Cool Slate (#E2E8F0)`，激活态深蓝 `Midnight Abyss (#000226)`。


* **阴影 (Shadows)**：**移除所有弥散投影**。
* *替代方案*：使用高对比度的边框颜色变化来表达层级。平面化 (Flat) 设计是绝对原则。



### **3.2 色彩逻辑：三色原则 (Tri-Chromatic Rule)**

严格遵守 PDF 中“不超过三个颜色”的限制 。

* **Canvas (画布)**：`#FFFFFF` (Absolute White) —— 绝不使用暖灰或米色。
* **Structure (结构)**：`#000000` (Structure Black) —— 用于文字、线条、Logo。
* **Signal (信号)**：`#000226` (Midnight Abyss) —— 唯一的品牌色，用于 CTA 按钮、Hover 边框、资讯栏背景。

### **3.3 按钮体系 (Buttons)**

* **Primary (实心)**：
* 背景：`#000226` (Midnight Abyss)。
* 文字：`#FFFFFF`。
* 形态：矩形，0px 圆角。


* **Secondary (线框)**：
* 背景：透明。
* 边框：`1px solid #000000`。
* **Hover 效果**：背景瞬间变为 `#000226`，文字变白。无过渡动画 (transition: none)，追求机械开关般的干脆感。



### **3.4 导航栏 (Navigation)**

* **形态**：不再悬浮透明，而是顶部固定实心白条，底部有一条 `1px` 黑线分割。
* **布局**：
* **左侧**：`FLEURLIS` Logo (衬线体，锐利)。
* **右侧**：功能性入口 —— `搜索`、`我的订单`、`预约`。使用纯文字或极简线条图标。

## **4. B2B 专属功能体验 (B2B UX)**

### **4.1 订单生成器：数字发票 (Digital Invoice)**

响应“截图微信沟通”的业务场景 。

* **设计目标**：生成的“订单确认页”必须看起来像一张打印好的、排版精美的 **A4 订货单**。
* **UI 布局**：
* **头部**：品牌 Logo + 当前日期 + 单号。
* **清单**：表格化布局 (Table Layout)，包含 `图片缩略图` | `款号` | `颜色` | `尺码` | `数量`。
* **底部**：备注栏 + 总件数大字展示。


* **截图优化**：
* 页面宽度适配手机竖屏。
* 背景纯白，字色纯黑，对比度极高（防止微信压缩后模糊）。
* 提供一个 **"生成长图 (Save as Image)"** 按钮，前端自动合成图片。



### **4.2 快速订货 (Quick Add)**

* 在列表页的每个商品卡片右下角，增加一个方形的 `[+]` 按钮。
* 点击不跳转详情页，而是弹出一个极简的 **Side Panel (侧边栏)** 选择尺码和颜色。
* **理由**：批发商通常是对照清单快速补货，不需要重复阅读详情页的“设计灵感”故事。

## **5. 交互与动效规范 (Interaction & Motion)**

### **5.1 动效原则：硬切 (Hard Cut)**

* **禁止**：缓动 (Easing)、淡入淡出 (Fade)、弹性 (Bouncing)。
* **提倡**：**Snap (瞬间吸附)**、**Toggle (切换)**。
* *理由*：B2B 用户需要的是响应速度。界面应像专业的交易终端一样，操作即反馈，无延迟。

### **5.2 图像滤镜 (CSS Filters)**

为了实现“日系冷剧”风格，所有非产品细节图（如氛围图、Banner）需应用全局 CSS 滤镜类：

```css
.filter-cold-drama {
  filter: contrast(95%) brightness(105%) saturate(80%) sepia(15%) hue-rotate(180deg);
  /* 提升青色调，压低暖色饱和度，保持高光通透 */
}

```

### **5.3 字体排版 (Typography)**

* 
**Display (标题)**: **Bodoni Moda** (All Caps) —— 极细的衬线，极高的粗细对比，传达 Gucci 式的权威感 。


* **Data (数据)**: **JetBrains Mono** 或 **DIN** —— 用于款号、价格、参数。赋予页面“工业说明书”的质感。
* **Body (正文)**: **Inter** —— 冷静、客观的中性字体。
